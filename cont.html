¿QUE ES GIT Y COMO FUNCIONA?

En el mundo del software, puedes llevar un historial, así como en un diario. No es algo que se crea en un día por arte de magia. El software tiene una vida, las aplicaciones tienen una vida y tienen versiones. En esa vida le añadimos nuevas funciones, corregimos errores, agregamos nuevas versiones. Para registrar el día a día de un proyecto se usa un sistema de control de versiones (VCS), un software que permite trabajar de forma colaborativa.

Cada desarrollador que está involucrado en el proyecto puede agregar los trabajos, cambios y correcciones que aporta al proyecto. Este sistema de control de versiones tiene un historial de todo lo que se ha hecho en el proyecto y quiénes lo han hecho. Este sistema de control de versiones es GIT.
